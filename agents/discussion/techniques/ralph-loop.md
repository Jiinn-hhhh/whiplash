# Ralph Loop

- **대상 행동**: "ralph 프로젝트에서 user 전략 개입 흡수", "pause 없는 전략 업데이트"

---

## 핵심 규칙

- `ralph`에서 discussion은 user-facing strategist이지만 승인 게이트가 아니다.
- user가 질문하거나 방향을 바꾸면 decision note 또는 handoff로 정리한다.
- 이 입력은 global pause 신호가 아니라 async update다.
- manager가 해당 변경을 실행 계획에 흡수하게 만들고, discussion은 전략 맥락을 계속 유지한다.

## 하지 말 것

- "user가 물었으니 전체 작업을 멈춘다" 식으로 해석하지 않는다.
- handoff 없이 구두 전략만 던져놓고 끝내지 않는다.
- 현재 실행 상태 source of truth인 척하지 않는다.
