# 분석 및 종합

- **대상 행동**: "자료 분석, 요약, 테마/패턴 추출"

---

## 1. 개별 자료 요약

### 절차
1. 원본 자료를 읽고 핵심 내용을 추출한다.
2. `workspace/teams/research/articles/`에 요약 문서를 작성한다.
   - 파일명: `YYYY-MM-DD-간략제목.md`
3. 요약 문서에 원본 위치를 반드시 명시한다.

### 요약 원칙: 지도 형태
- 요약은 백과사전이 아닌 **지도**다.
- 핵심만 추출하고, 디테일은 원본 참조로 연결한다.
- "이 논문은 X를 다룬다. 핵심 기여는 Y이다. 상세는 원본 §3 참조."

---

## 2. 테마/패턴 추출

### 절차
1. 여러 자료를 관통하는 공통 주제/패턴을 식별한다.
2. `workspace/teams/research/themes/`에 테마별 문서를 작성한다.
3. 각 테마에 해당하는 자료들의 출처를 명시한다.

### 테마 문서 구조

```markdown
# 테마: [테마명]

- **Date**: YYYY-MM-DD
- **Sources**: [관련 자료 목록과 위치]

## 개요
이 테마가 무엇인지 (2-3문장)

## 주요 발견
- 발견 1 (출처: source-A)
- 발견 2 (출처: source-B)

## 현재 접근 vs 대안
| 관점 | 현재 접근 | 대안 | 근거 |
|------|----------|------|------|
| ... | ... | ... | 출처 |

## 시사점
이것이 우리 프로젝트에 의미하는 바
```

---

## 3. 비교 기반 분석

단순 요약에 그치지 않고, **"현재 접근 vs 대안"** 비교를 통해 인과적 인사이트를 추출한다.

### 절차
1. 현재 우리의 접근 방식을 명확히 정의한다.
2. 자료에서 발견한 대안적 접근을 정리한다.
3. 각 접근의 장단점을 비교한다 (조건, 효과, 한계).
4. 비교에서 도출된 인사이트를 기록한다.
5. 행동 가능한 제안이 있으면 별도로 정리한다.

### 원칙
- 비교 없는 요약은 가치가 낮다. 항상 "그래서 우리한테 어떤 의미인가?"를 포함한다.
- 비교 시 근거를 명시한다 (어떤 논문에서, 어떤 데이터에 근거하여).

## 4. 기존 지식과의 연결

- 새로운 분석 결과를 기존 `memory/researcher/` 메모와 대조한다.
- 기존 교훈을 강화하거나 반박하는 증거가 있으면 명시한다.
- 연결점을 발견하면 해당 교훈 번호를 참조한다 (`LESSON-NNN`).
