# 자료 수집

- **대상 행동**: "연구 자료(논문, 기사, 데이터) 수집 및 원본 아카이빙"

---

## 수집 대상

- 학술 논문 (arXiv, ACL Anthology, IEEE, etc.)
- 기술 기사 / 블로그 포스트
- 공식 문서 (프레임워크, 라이브러리)
- 데이터셋 및 벤치마크
- 코드 저장소 (GitHub 등)

---

## 절차

1. **검색**: 서브에이전트를 활용하여 키워드/주제 기반 검색을 수행한다.
   - 검색 전략, 탐색 범위, 순서는 자율적으로 판단한다.
2. **선별**: 관련성과 신뢰도를 기준으로 자료를 선별한다.
   - 학술 논문: 인용 수, 발표 장소, 저자 신뢰도
   - 기사/블로그: 저자 배경, 구체적 근거 포함 여부
3. **확보** (아래 수집 후 처리 체크리스트를 따른다)
4. **원본 저장**: `workspace/teams/research/sources/`에 원본을 저장한다.
   - PDF, 텍스트 파일, 스크린샷 등 원본 형태 유지
   - 파일명: `YYYY-MM-DD-간략제목.{확장자}`
5. **메타데이터 기록**: 원본과 함께 메타데이터를 기록한다.

---

## 수집 후 처리 체크리스트

소스를 발견했을 때, 아래를 순서대로 확인한다. 검색 전략이나 탐색 범위는 자율이지만, 찾은 자료의 **확보와 기록 품질**은 이 체크리스트로 보장한다.

### 1. DOI 확인
- 학술 논문이면 DOI를 확인하고 기록한다.
- DOI는 논문의 고유 식별자. 나중에 원본 재접근, 인용, 메타데이터 조회의 기준점이 된다.
- DOI가 없는 자료(블로그, 비공식 문서 등)는 URL을 기준으로 한다.

### 2. 원본 확보 시도
- 전문(full text)에 접근 가능하면 그대로 저장한다.
- 접근 불가(paywall 등)하면 Open Access 버전을 탐색한다:
  - 저자의 개인 사이트 / 기관 레포지토리
  - arXiv, PubMed Central 등 프리프린트 서버
  - 출판사의 OA 옵션
- OA도 없으면 초록(abstract) + 메타데이터를 기록하고, 원본 미확보임을 명시한다.

### 3. 메타데이터 기록
- 아래 필드를 빠짐없이 기록한다 (해당하는 것만):
  - 제목, 저자, 연도, 출처(학회/저널/사이트)
  - DOI (있는 경우)
  - URL (원본 접근 경로)
  - 원본 확보 여부 (전문 / 초록만 / OA 버전)

### 4. 아카이빙
- 원본 파일을 sources/에 저장한다. **요약만 남기고 원본을 버리지 않는다.**
- 같은 논문의 여러 버전(프리프린트 vs 출판본)이 있으면 출판본을 우선한다.

> **원칙**: 검색은 자유롭게, 확보는 빈틈없이.

---

## 메타데이터 구조

원본 파일과 같은 위치에 `.md` 파일로 메타데이터를 기록한다.

```markdown
# [자료 제목]

- **Type**: paper | article | docs | dataset | repo
- **Authors**: [저자/출처]
- **Date**: YYYY-MM-DD (원본 발행일)
- **DOI**: [DOI 또는 "N/A"]
- **URL**: [원본 링크]
- **Full-text**: yes | abstract-only | OA-version
- **Retrieved**: YYYY-MM-DD (수집일)
- **Tags**: `tag1`, `tag2`

## 요약 (1-2문장)
이 자료가 다루는 핵심 내용

## 수집 이유
어떤 연구 질문/작업과 관련되어 수집했는지
```

---

## 원칙

- **원본 보존**: 요약만 남기지 않는다. 원본을 항상 저장한다.
- **출처 추적 가능**: 메타데이터에서 원본까지 추적 가능해야 한다.
- **중복 방지**: 저장 전 기존 sources/에 동일 자료가 있는지 확인한다.
- **서브에이전트 활용**: 검색, 다운로드 등 반복 작업은 서브에이전트에게 위임한다.
