# 지식 기여

- **대상 행동**: "교훈 작성, 공유 공간 기여"

---

## 1. 교훈 추출

### 시점
- 연구 리포트 완료 후
- 실험 종료 후 (성공/실패 무관)
- 테마 분석에서 일반화 가능한 패턴 발견 시

### 절차
1. 연구 결과에서 다른 작업/프로젝트에도 적용 가능한 인사이트를 식별한다.
2. 기존 `memory/knowledge/lessons/`의 교훈과 의미적 중복 여부를 확인한다.
   - 중복이면: Manager에게 기존 교훈 보강을 제안한다.
   - 새로운 것이면: `common/formats.md`의 교훈(Lesson) 양식으로 작성한다.
3. Manager와의 디스커션을 통해 교훈의 일반성과 정확성을 검증한다.
4. 검증된 교훈을 `memory/knowledge/lessons/`에 기록한다.

---

## 2. Cross-branch Transfer

다른 분야/프로젝트의 교훈을 현재 연구에 적용한다.

### 절차
1. 새 연구를 시작할 때 `memory/knowledge/lessons/`를 검토한다.
2. 직접 관련되지 않더라도 구조적으로 유사한 교훈을 탐색한다.
   - 예: 이미지 분류 실험의 교훈이 텍스트 분류에도 적용 가능한가?
3. 적용 가능한 교훈을 발견하면 현재 연구에 어떻게 적용할지 기술한다.

### 원칙
- 교훈은 발생한 분야에만 갇히지 않는다. 구조적 유사성을 적극적으로 탐색한다.
- 전이 적용 시 원래 맥락과 현재 맥락의 차이를 명시한다.

---

## 3. 산출물 공유

### 팀 내부 → 공유 공간 이동 절차

1. `workspace/teams/research/` 내부에서 산출물을 작성하고 정리한다.
2. 확정된 산출물만 공유 대상으로 선정한다.
   - 확정 기준: 분석 완료, 출처 검증 완료, 자체 검토 완료
3. 산출물 유형에 따라 이동한다:

| 산출물 | 이동 위치 |
|--------|-----------|
| 교훈 | `memory/knowledge/lessons/` |
| 리서치 리포트 (최종) | `memory/knowledge/docs/` 또는 `workspace/shared/` |
| 다른 팀 요청 응답 | `workspace/shared/` |

4. 중간 결과, 초안, 미검증 분석은 `workspace/teams/research/`에 남긴다.

---

## 4. 정보 요청 응답

다른 팀이 정보를 요청할 때:

1. `workspace/shared/`에서 요청을 확인한다.
2. 이미 보유한 정보면: 해당 자료/분석을 `workspace/shared/`에 게시한다.
3. 추가 조사가 필요하면: 예상 소요를 알리고 조사를 진행한다.
4. 응답 시 출처와 신뢰도를 명시한다.
