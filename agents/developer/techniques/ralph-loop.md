# Ralph Loop

- **대상 행동**: "승인 대기 없는 구현/검증 반복", "끝까지 결과물 만들기"

---

## 기본 사이클

`map -> verify/debug -> implement -> test -> review -> improve`

- user 확인을 기다리며 멈추지 않는다.
- blocker를 만나면 manager에게 이유와 fallback을 짧게 알리고, 가능한 대체 경로로 계속 진행한다.
- reviewer/test-automator/debugger 같은 specialist를 적극 활용해 한 사이클 안에서 검증까지 닫는다.

## 종료

- manager가 현재 랄프 종료 정책을 만족했다고 판단할 때만 task를 닫는다.
- "대충 괜찮아 보임"으로 스스로 끝내지 않는다.
