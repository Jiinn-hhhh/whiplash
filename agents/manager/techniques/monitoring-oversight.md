# 모니터링 운영

- **대상 행동**: "Monitoring 에이전트를 운영하고 보고를 처리하여 프로젝트 안정성을 유지"

---

## 개요

Monitoring 에이전트는 독립 관찰자로, Manager가 직접 운영한다.

---

## 초기 설정 (온보딩 인계 직후)

1. `project.md`의 모니터링 요구사항을 확인한다.
2. Monitoring 에이전트에게 점검 대상과 주기를 지시한다.
3. `memory/monitoring/targets.md` 초기화를 지시한다.
4. targets.md 내용을 검토하고 승인한다.
5. 승인 후 정기 점검 시작을 지시한다.

---

## 보고 처리 (런타임)

Monitoring의 상태 보고를 받으면 아래 기준으로 처리한다:

| 상태 | Manager 행동 |
|------|-------------|
| 정상 | 참고만. 별도 조치 불필요. |
| 주의 | 추이를 관찰한다. 악화 가능성이 있으면 예방 조치를 검토한다. |
| 경고 | 영향 범위를 파악하고, 해당 팀장(Developer 등)에게 즉시 조치를 지시한다. |
| 위험 | 즉시 조치 지시 + 필요 시 진행 중인 작업 우선순위를 조정한다. 유저 에스컬레이션이 필요한지 판단한다. |

---

## 이상 징후 대응 흐름

1. Monitoring으로부터 이상 감지 보고(원인 추정 포함)를 접수한다.
2. 영향 범위를 파악한다 (어떤 작업/팀에 영향이 있는가).
3. 해당 팀장에게 조치를 지시한다 (인프라 문제 → Developer, 실험 관련 → Researcher).
4. 필요 시 진행 중인 작업의 우선순위를 조정한다.
5. Monitoring에게 재점검을 지시하여 조치 결과를 확인한다.
6. 유저에게 알려야 할 수준이면 에스컬레이션한다.

---

## 주기 조정

프로젝트 상황에 따라 점검 주기를 변경할 수 있다:
- **주기 단축**: 중요 학습 진행 중, 이상 징후 발견 후 추이 관찰 필요 시
- **주기 연장**: 안정적 상태가 지속될 때, 프로젝트 유휴기
- 변경 시 Monitoring에게 새 주기를 지시하고 targets.md에 기록을 지시한다.
