# 문서 양식 템플릿

모든 에이전트가 공유하는 표준 양식.
통일된 형식으로 작성해야 에이전트 간 산출물을 쉽게 파싱할 수 있다.

---

## 1. 교훈 (Lesson)

```markdown
# LESSON-NNN: [제목]

- **Date**: YYYY-MM-DD
- **Author**: [에이전트명]
- **Source**: [어떤 작업/토론에서 도출되었는지]
- **Tags**: `tag1`, `tag2`

## 교훈
> 핵심 내용 1-2문장

## 근거
구체적인 경험/데이터 기반 설명

## 적용 가이드
이 교훈을 언제/어떻게 적용해야 하는지
```

---

## 2. 토론 (Discussion)

```markdown
# DISC-NNN: [주제]

- **Date**: YYYY-MM-DD
- **Initiated by**: [에이전트명]
- **Participants**: [관련 에이전트/팀]
- **Status**: open | resolved

## 주제
> 토론하고자 하는 질문/이슈

## 논의

### [에이전트명] — YYYY-MM-DD
내용 (append-only)

### [에이전트명] — YYYY-MM-DD
응답 (append-only)

## 결론
(resolved 시 작성)
```

---

## 3. 회의록 (Meeting)

```markdown
# MEET-NNN: [회의 주제]

- **Date**: YYYY-MM-DD
- **Called by**: [에이전트명]
- **Participants**: [참석자]

## 안건
> 회의 목적

## Round 1: 입장
### [에이전트명]
입장 서술

### [에이전트명]
입장 서술

## Round 2: 응답
### [에이전트명]
다른 입장에 대한 응답

### [에이전트명]
다른 입장에 대한 응답

## Round 3: 종합
### [주최자]
종합 및 결론

## Action Items
- [ ] 누가 / 뭘 할 것인지
```

---

## 4. 보고서 (Report)

```markdown
# [보고서 제목]

- **Date**: YYYY-MM-DD
- **Author**: [에이전트명]
- **For**: [보고 대상]
- **Tags**: `tag1`, `tag2`

## 요약
- **무엇**: 이 보고서가 다루는 주제
- **핵심 발견**: 주요 결론/결과 (2-3문장)
- **시사점**: 이것이 의미하는 바 / 다음 단계

## 내용
본문 (근거 포함)

## 참고한 교훈
- Cite LESSON-NNN: 어떻게 적용했는지

## 다음 단계
- 후속 작업/권장 사항
```

---

## 요약 수준 가이드라인

| 어디에서 | 요약 수준 | 이유 |
|---------|----------|------|
| index.md | 항목당 1-2줄 | 지도. "이런 게 있다" 수준 |
| 교훈 | 핵심 1-2문장 + 근거/적용가이드 별도 섹션 | 핵심은 짧되 디테일은 섹션으로 |
| 보고서 요약 | 구조화된 5-10줄 (What/Finding/Implication) | 정보 손실 없이 구조적 |
| 토론/회의록 | 제한 없음 | 원본 기록 |
