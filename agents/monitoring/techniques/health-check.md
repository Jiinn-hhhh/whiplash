# 건강 상태 점검

- **대상 행동**: "Manager 지시에 따라 정해진 항목을 점검하고 현재 상태를 기록"

---

## 1. 점검 절차

1. `memory/monitoring/targets.md`에서 점검 대상 목록을 확인한다.
2. 항목별로 현재 값을 수집한다.
3. 현재 값과 임계치를 비교하여 상태 등급을 판정한다.
4. 상태 보고서를 작성한다.
5. 점검 이력을 `memory/monitoring/`에 기록한다.
6. 경고/위험 항목이 있으면 즉시 보고 절차(`alert.md`)를 따른다.

---

## 2. 점검 항목 분류

| 분류 | 항목 예시 |
|------|-----------|
| 시스템 | CPU 사용률, RAM 사용량, 디스크 사용량 |
| 인프라 | 서버 상태, 컨테이너 상태, 네트워크 연결 |
| GPU | GPU 사용률, GPU 메모리, GPU 온도 |
| 애플리케이션 | 프로세스 생존 여부, 에러율, 응답 시간 |
| 저장소 | 디스크 잔여 용량, 체크포인트 크기, 로그 크기 |

실제 점검 항목은 `targets.md`에 프로젝트별로 정의된다. 위 분류는 참고용이다.

---

## 3. 상태 등급

| 등급 | 기준 | 행동 |
|------|------|------|
| 정상 | 임계치 80% 미만 | 기록만 |
| 주의 | 임계치 80% 이상 ~ 위반 미만 | 보고서에 표시, 추이 주시 |
| 경고 | 임계치 위반 | 즉시 보고 |
| 위험 | 복수 항목 위반 또는 서비스 중단 | 즉시 보고, 최우선 표시 |

---

## 4. 상태 보고서 양식

```markdown
# 상태 보고서

- **Date**: YYYY-MM-DD HH:MM
- **From**: Monitoring
- **To**: Manager
- **전체 상태**: 정상 / 주의 / 경고 / 위험

## 점검 결과

| 항목 | 현재 값 | 임계치 | 상태 |
|------|---------|--------|------|
| ... | ... | ... | 정상/주의/경고/위험 |

## 이상 항목 상세
(경고/위험 항목이 있을 때만)

### [항목명]
- **현재 값**: ...
- **임계치**: ...
- **추이**: 최근 N회 점검 값 나열
- **원인 추정**: 패턴 분석 기반 추정 (처방은 포함하지 않음)

## 추이 요약
전회 대비 변화, 지속적 추세 등
```

## 5. 모니터링 대상 관리

### targets.md

모니터링 대상은 `memory/monitoring/targets.md`에서 관리한다.

```markdown
# 모니터링 대상

## 점검 항목

| 항목 | 임계치 | 점검 주기 | 비고 |
|------|--------|-----------|------|
| ... | ... | ... | ... |

## 변경 이력

| 날짜 | 변경 내용 | 승인 |
|------|-----------|------|
| ... | ... | Manager |
```

### 운영 규칙

- 초기 항목은 프로젝트 시작 시 Manager 지시로 설정한다.
- 항목 추가/삭제, 임계치 변경은 Manager 승인이 필요하다.
- Monitoring이 필요하다고 판단한 변경은 Manager에게 제안하고 승인을 받는다.
- 변경 시 `targets.md`의 변경 이력에 기록한다.
