# Domain: Deep Learning

딥러닝 프로젝트에서 모든 에이전트가 공유하는 배경 지식.

---

## 핵심 개념

### 학습 파이프라인
데이터 수집 → 전처리 → 모델 설계 → 학습 → 평가 → 배포. 각 단계가 다음 단계의 품질을 결정한다.

### 실험 관리
- 모든 실험은 재현 가능해야 한다 (코드 버전, 데이터 버전, 하이퍼파라미터 기록).
- 실험 결과는 정량적 지표로 비교한다.
- 실패한 실험도 기록한다 — 무엇이 안 되는지 아는 것도 지식이다.

### 데이터 중심
모델 아키텍처보다 데이터 품질이 더 큰 영향을 미치는 경우가 많다. 데이터 수집, 정제, 증강에 충분한 시간을 투자한다.

---

## 용어

| 용어 | 설명 |
|------|------|
| Epoch | 전체 데이터셋을 한 번 순회 |
| Batch size | 한 번의 업데이트에 사용하는 샘플 수 |
| Learning rate | 파라미터 업데이트 크기를 제어하는 값 |
| Overfitting | 학습 데이터에는 좋지만 새 데이터에 일반화하지 못하는 상태 |
| Loss | 모델 예측과 정답 사이의 차이를 수치화한 값 |
| Checkpoint | 학습 중간 상태를 저장한 스냅샷 |
| Fine-tuning | 사전학습된 모델을 특정 작업에 맞게 추가 학습 |
| Inference | 학습된 모델로 새 데이터에 대해 예측을 수행 |

---

## 품질 기준

### 실험
- 재현 가능성: 같은 설정으로 같은 결과를 얻을 수 있는가?
- 비교 가능성: 베이스라인과 동일한 조건에서 비교했는가?
- 통계적 유의성: 단일 실행이 아닌 여러 시드에서 확인했는가?

### 모델
- 성능 지표: 정확도, F1, BLEU 등 과제에 적합한 메트릭 사용
- 효율성: 학습 시간, 추론 속도, 메모리 사용량
- 일반화: 학습 데이터와 다른 분포의 데이터에서의 성능

### 코드
- 설정 분리: 하이퍼파라미터를 코드에 하드코딩하지 않음
- 로깅: 학습 과정의 주요 지표를 기록
- 체크포인트: 학습 중단 시 이어서 학습 가능
