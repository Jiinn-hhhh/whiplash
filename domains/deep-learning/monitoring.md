딥러닝 도메인에서 Monitoring에게 추가되는 지침. 기본 `agents/monitoring/profile.md`를 **보충**한다.

---

## DL 인프라의 특성

딥러닝 프로젝트에서 인프라 모니터링이 특별히 중요한 이유:

- **GPU가 핵심 자원이다.** GPU가 놀고 있으면 비용과 시간이 낭비된다. GPU 사용률은 가장 중요한 모니터링 지표다.
- **학습이 장시간 지속된다.** 수 시간~수 일 동안 돌아가는 학습 프로세스가 중간에 죽으면 전체 시간이 낭비된다. 프로세스 생존 여부 확인이 중요하다.
- **체크포인트가 디스크를 소모한다.** 주기적으로 저장되는 체크포인트 파일이 디스크를 빠르게 채울 수 있다. 디스크 사용량 추이를 주시해야 한다.
- **데이터 파이프라인이 병목이 될 수 있다.** GPU가 데이터를 기다리는 동안 놀게 되면, GPU 문제가 아니라 데이터 로딩 문제다.

딥러닝 프로젝트에서 `targets.md` 초기 설정 시 참고하는 항목:

| 항목 | 임계치 예시 | 비고 |
|------|-------------|------|
| GPU 사용률 | 0% 지속 시 경고 | 패턴 분석 필요 (아래 참조) |

| 학습 프로세스 생존 여부 | 프로세스 미존재 시 경고 | 학습 중일 때만 적용 |

## GPU 사용률 패턴과 원인 추정

GPU 사용률은 단일 수치보다 **패턴**이 중요하다. 아래는 흔한 패턴과 추정 가이드:

### 안정적 높은 사용률 (90%+)
- **상태**: 정상
- **의미**: GPU가 효율적으로 활용되고 있음

### 주기적 0% ↔ 90%+ 반복
- **상태**: 주의
- **원인 추정**: 데이터 로딩 병목일 수 있음. GPU가 다음 배치를 기다리는 동안 놀게 되는 패턴.
- **기록**: 0% 구간의 길이와 주기를 기록

### 지속적 0%
- **상태**: 경고
- **원인 추정**: 학습 프로세스가 종료되었거나 에러로 멈춘 상태일 수 있음. 또는 프로세스가 GPU를 점유만 하고 실제 연산을 하지 않는 상태일 수 있음.
- **기록**: 언제부터 0%인지, 프로세스 상태와 교차 확인

### 점진적 하락 (90% → 70% → 50% ...)
- **상태**: 주의
- **원인 추정**: 메모리 누수, 데이터 파이프라인 성능 저하, 또는 워크로드 특성 변화(에폭 후반 데이터 크기 변화 등)일 수 있음.
- **기록**: 하락 시작 시점, 하락 속도, 동시기 다른 지표 변화

### 간헐적 스파이크/드롭
- **상태**: 기록
- **원인 추정**: 체크포인트 저장, 평가(validation) 실행, 외부 프로세스 간섭 등 일시적 요인일 수 있음.
- **기록**: 스파이크/드롭 시점과 다른 이벤트(체크포인트 저장 등)의 상관관계

**원칙**: 패턴을 기록하고 원인을 추정하되, 처방("num_workers를 늘려라", "prefetch를 켜라" 등)은 하지 않는다.

## 다중 GPU 환경

다중 GPU 환경에서는 개별 GPU를 각각 점검한다.

- **개별 점검**: GPU 0, GPU 1, ... 각각의 사용률, 메모리, 온도를 기록한다.
- **불균형 감지**: GPU 간 사용률 차이가 크면(예: GPU 0은 90%, GPU 1은 10%) 불균형으로 기록한다.
- **불균형 원인 추정**: 데이터 병렬화 문제, 모델 병렬화 불균형, 또는 특정 GPU만 사용하는 설정 오류일 수 있음.

## Researcher와의 경계

| 영역 | Monitoring | Researcher |
|------|-----------|-----------|
| GPU 사용률, 메모리, 온도 | O | X |
| 프로세스 생존 여부 | O | X |
| 디스크 사용량 | O | X |
| loss 수렴 여부 | X | O |
| 오버피팅 판단 | X | O |
| 실험 지표 해석 | X | O |
| 학습 속도(iter/sec) | O (인프라 관점) | O (실험 관점) |

**학습 속도**는 양쪽 모두 볼 수 있지만 관점이 다르다:
- Monitoring: "iter/sec가 떨어지고 있다, GPU 사용률 하락과 상관관계가 있다" (인프라 관점)
- Researcher: "iter/sec가 떨어지고 있다, 배치 크기 변경의 영향일 수 있다" (실험 관점)
