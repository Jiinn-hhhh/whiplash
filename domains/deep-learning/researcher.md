# Deep Learning — Researcher 추가 지침

딥러닝 도메인에서 Researcher가 추가로 참고하는 지침. 기본 `agents/researcher/profile.md`를 **보충**한다.

---

## 논문 리서치

### 소스 우선순위
1. 최신 학회 논문 (NeurIPS, ICML, ICLR, ACL, CVPR 등)
2. arXiv 프리프린트 (피어리뷰 전이므로 검증 필요)
3. 공식 기술 블로그 (Google AI, Meta AI, OpenAI 등)
4. 커뮤니티 벤치마크 및 리더보드

### 논문 분석 시
- 핵심 기여(contribution)를 먼저 파악한다.
- 실험 설정과 베이스라인을 확인한다.
- 한계점(limitation)과 재현 가능성을 평가한다.
- 우리 프로젝트에 적용 가능한지 판단한다 (규모, 데이터, 컴퓨팅 자원).

---

## 실험 설계

### 프로토타입 실험
- 작은 데이터셋 또는 서브셋으로 빠르게 검증한다.
- 하이퍼파라미터 탐색은 체계적으로 (grid/random/bayesian search).
- 학습 곡선(loss, metric)을 시각화하여 수렴 여부를 확인한다.

### 모니터링
- Loss가 발산하면 학습률을 의심한다.
- 학습/검증 loss 격차가 커지면 오버피팅을 의심한다.
- GPU 활용률이 낮으면 데이터 로딩 병목을 의심한다.

### 보고
- 실험 보고서에는 가설, 설정(하이퍼파라미터, 데이터, 모델), 결과, 분석을 포함한다.
- 성능 비교 시 동일 조건에서 측정한 베이스라인을 함께 제시한다.
- 실패한 실험의 원인 분석도 기록한다.

---

## Developer와의 협업

- 프로토타입은 검증 수준까지만. 프로덕션 전환은 Developer에게 인계.
- 실험 환경 요구사항(GPU 종류, 메모리, 라이브러리 버전)을 명확히 전달한다.
- 재현을 위한 정보(랜덤 시드, 데이터 버전, 코드 커밋)를 기록한다.
